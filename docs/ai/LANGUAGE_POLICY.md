# Vault Language Policy

## 基本方針

External Memory Vault に保存する自然文は、原則として日本語で書く。

理想は、メモ全体の自然文のうち 90% 以上が日本語である状態。これは品質目標であり、機械的に失敗扱いする必須条件ではない。

## 英語を許容するもの

次のような単語レベルの英語はそのままでよい。

- ファイル名、ディレクトリ名、URL、コマンド、ログ断片。
- プログラムの変数名、関数名、クラス名、設定キー。
- 製品名、ライブラリ名、MCP tool 名、GitHub Actions の job 名。
- `memory proposal`、`session summary`、`canonical` など、短い専門用語として扱う語。
- 引用やエラーメッセージ。ただし必要なら日本語で要約を添える。

## 避けるもの

- 英文の段落。
- 英文だけの説明文。
- テンプレートの説明欄を英語のまま埋めること。
- 英語の箇条書きが続き、日本語の説明がない状態。

## 書き方

- 見出しは日本語を基本にする。
- frontmatter の key や enum 値は既存ツールとの互換性を優先して英語のままでよい。
- evidence にコマンドやファイルパスを書く場合、その下に日本語で「何を確認したか」を書く。
- 外部資料の英文を貼る代わりに、日本語で要約し、必要な範囲だけ短く引用する。
- 判断や提案を書くときは、未来の agent がそのまま行動できる具体性を持たせる。

## レビュー観点

- 英文を読まなくても内容が分かるか。
- 英語は単語、識別子、ログ、引用に収まっているか。
- 日本語の説明が主で、英語は補助になっているか。
- secrets、顧客情報、個人情報が含まれていないか。
