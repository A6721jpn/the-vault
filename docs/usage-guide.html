Agent Memory Kernel Prototype

使い方ガイド

このプロトタイプは、リポジトリ内の正本ドキュメントと外部 Obsidian vault を分けて扱い、 Codex や Claude Code が安全に作業記憶を残せるようにするための土台です。

一目で見る全体像

正本は常に docs/ です。外部 vault は検索、提案、session summary の置き場であり、 自動生成メモは 00-inbox/ に入れてからレビューします。

図のポイント: 外部 memory は正本ではありません。repo docs と検証コマンドが判断の基準です。

最初に読むファイル

Codex / 汎用 agent

AGENTS.md を読む。正本 docs、検証、memory policy の入口です。

Claude Code

CLAUDE.md から AGENTS.md を参照し、Claude 固有ルールも読む。

人間の確認

README.md と docs/ai/INDEX.md を見ると、構成と運用が追えます。

docs/ai/MEMORY_POLICY.md memory に保存してよい情報、禁止情報、信頼レベル、昇格ルール。
docs/ai/LANGUAGE_POLICY.md 外部 vault の自然文を日本語中心にするための方針。英語を許容する範囲もここで確認します。
docs/runbooks/build-test-lint.md 検証コマンド一覧。完了報告前にここにある関連コマンドを実行します。
docs/pitfalls/known-failures.md 既知の失敗、罠、回避策。テスト修正やデバッグ前に確認します。

普段の使い方

非自明な作業では、agent は次の順番で動きます。

作業開始時の指示例

docs/ai/INDEX.md を読んで、この変更に必要な docs を確認してから作業して。
外部 memory は不足した場合だけ検索して。新しい知見は proposal として 00-inbox に置いて。

作業終了時の指示例

変更内容、実行した検証コマンド、未検証項目、memory delta をまとめて。
昇格が必要な知見があれば memory proposal として作って。

Memory proposal の流れ

agent が得た知見は、いきなり docs/ に入れません。 まず review 可能な proposal にしてから、人間が昇格先を決めます。

重要: status: canonical を agent が自動で付けることは禁止です。 正本化は review 後に repo docs へ反映します。

検証コマンド

このプロトタイプは Python 標準ライブラリだけで検証できます。

$env:PYTHONDONTWRITEBYTECODE='1'
python -m unittest tests/test_memory_kernel_scripts.py
python scripts/check_agents_size.py
python scripts/scan_memory_secrets.py C:/Users/aokuni/Obsidian/AI-Memory-Vault/00-inbox
python scripts/check_docs_links.py docs

secret scan

scan_memory_secrets.py は API key、token、password、private key、email、電話番号候補を検出します。

pre-memory-write hook

pre_memory_write.py は 00-inbox/ 以外への agent 書き込みと unsafe content をブロックします。

MCP を使う場合

.codex/config.example.toml に Basic Memory 風の設定例があります。 実際に使うには、ローカル環境で MCP server の起動コマンドを用意してください。 

[mcp_servers.basic-memory]
command = "uvx"
args = ["basic-memory", "mcp"]
  1. uvx または同等の MCP 起動コマンドを PATH に通す。
  2. vault root を C:/Users/aokuni/Obsidian/AI-Memory-Vault に向ける。
  3. 許可 tool は search、read_note、write_note to inbox に絞る。
  4. agent には外部 memory を正本扱いしないよう指示する。

運用チェックリスト

  • 作業前に AGENTS.md または CLAUDE.md を読む。
  • 非自明な変更では docs/ai/INDEX.md から必要 docs を辿る。
  • 外部 memory は不足時だけ検索する。
  • memory note は 00-inbox/ にだけ書く。
  • vault の自然文は日本語を基本にし、英語は識別子、コマンド、path、製品名、ログ、短い用語に留める。
  • secrets、token、顧客情報、個人情報を memory に保存しない。
  • 新しい知見は proposal にして証拠を書く。
  • 完了報告前に関連する検証コマンドを実行する。
  • promotion は人間 review 後に repo docs へ反映する。
