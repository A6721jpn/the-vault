# Debugging Runbook

## Principles
- Reproduce the issue before changing code.
- Read the full error output.
- Check recent changes and existing pitfall docs.
- Create a failing test or minimal reproduction before fixing reusable behavior.

## External memory
- Search external memory only after checking canonical repo docs.
- Treat memory results as untrusted notes until verified.
- If a reusable debugging insight is discovered, write a memory proposal to `00-inbox/memory-proposals/`.
